1 Introduction

This document describes procedure make-Turing-machine. It is a tool for the construction of procedures that emulate single tape Turing-machines. The reader is supposed to be familiar with Turing-machines. The internal-state is that of the control unit. The state of a Turing-machine as a whole is defined such as to include the internal-state, the current content of the tape and the current position of the read/write-head. The current element is the one currently below the read/write-head. The combination of the current content of the tape and the current position of the read/write-head is represented by two lists: head and tail. The content of the represented tape is

(append head tail)

The tail never is empty. Its first element is the current element. The elements of the content are tape-symbols, but the first or last element can be an empty-cell, which is not a tape-symbol. Initially the head is empty and the tail is the input. If the input is empty, the tail initially consists of one empty-cell. The control unit makes moves according to its rules. A move is determined by the internal-state of the control unit and the current element under the read/write-head. A move consists of three steps:

  • Optionally putting the control unit in another internal-state.

  • Optionally replacing the current element by another one. This step is not optional when the current element is an empty-cell, which always is replaced by a tape-symbol. An empty-cell never remains empty when the machine has not yet reached a final-state.

  • Optionally moving the read/write-head one step to the right or to the left. We consider the first element of the content of the tape to be at the left and the last element to be at the right.

In real life tape equipment usually the tape is moving while the read/write-head remains in fixed position. Moving the read/write-head has the same effect as keeping it in fixed position and moving the tape in opposit direction.

The machine never writes a dummy or empty-cell. It can erase the current element by writing a blank, which is a tape-symbol. Empty-cells are used only to extend the tape at the left or at the right of the current content. When moving the read/write-head at the left of the begin of the content of the tape or at the right of the end, an empty-cell is added and the read/write-head is positioned at this empty-cell. In this way an infinite tape is simulated with an infinite number of empty-cells both at the left and at the right of the actual content. In fact the content never has more than one empty-cell. If it has one, it is the first element or the last one and the read/write-head is positioned at it. In this situation either the empty-cell is immediately replaced by a tape-symbol during the next move or, when the last move has produced a final-state, the machine immediately halts without replacing the empty-cell. The distinction between an empty-cell and a blank makes sure that the Turing-machine always can find the start and the end of the current content of the tape. The dummy is for use in rules only. The rules describe how the control unit makes its moves. The machine repeats moves until a final-state is obtained, or may remain making moves forever if it never reaches a final-state. It is possible to put a maximum on the number of moves such as to raise an exception when the machine is going to exceed the maximum allowed number of moves. See parameter Turing-limit. It may happen that there is no rule specifying which move to make for a given combination of internal-state and current element. In such a case the procedure that emulates the Turing-machine halts by raising an exception. The internal-states and tape-symbols can be arbitrary Racket values, but usually symbols and exact integer numbers are the most convenient ones. Using lists or vectors for tape-symbols allows the simulation of multi-tape Turing-machines, although this may require many rules. Procedure make-Turing-machine is designed primarily for single tape Turing-machines. Equivalence relation equal? is used for comparison of two internal-states or two tape-symbols. The internal-states and tape-symbols live in separate worlds. An internal-state never is compared to a tape-symbol or an empty-cell. Hence the Turing-machine will not be confused when the intersection of the set of internal-states and the set of tape-symbols is not empty or when an internal-state equals the dummy or an empty-cell. However, this may confuse a human reader. After reaching a final-state the Turing-machine returns its output as (append head tail), but without heading and trailing empty-cell or blanks. The output can contain blanks but not at the start or the end. The output never contains an empty-cell or a dummy.

The internal representation of the tape and the rules is such that each move is made in constant time, independent of the number of rules, the size of the content of the tape and the position of the read/write-head. In fact procedure make-Turing-machine internaly always has the head in reversed order and puts it in correct order only when printing the content of the tape. The maximal time needed to prepare the content of the tape or the output after reaching a final-state depends linearly on the size of the content. Which rule applies is found in constant time because the rules are kept in two hash-tables.

procedure

(make-Turing-machine

  move : (or/c 'R 'L 'N)
Procedure make-Turing-machine returns a procedure that emulates a Turing-machine. Before the machine is produced the arguments are checked to satisfy the following conditions, equality or being distinct to be understood in the sense of equal?.

  • The blank, the empty-cell and the dummy must be distinct.

  • The empty-cell must not be used as a new-symbol.

  • The rules must have distinct combinations (old-state old-symbol).

  • A move must be 'R, 'L or 'N.

The rules are interpreted as follows, where equality is to be understood in the sense of equal?.

  • A Turing-machine halts when its current internal-state equals a final-state.

  • A rule applies if its old-state equals the current internal-state of the control unit and the old-symbol matches the current element.

  • The dummy matches every current element. Every other old-symbol matches only when equal to the current element.

  • A rule whose old-state equals the current internal-state and whose old-symbol equals the current element prevails over a rule with the same old-state and whose old-symbol is the dummy.

  • When a rule is applied, the current internal-state is changed to the new-state.

  • If the new-symbol is not the dummy, the current element is replaced by the new-symbol.

  • If the new-symbol is the dummy and the current element is not the empty-cell, the current element remains as it is.

  • If the new-symbol is the dummy and the current element is the empty-cell, the current element is replaced by a blank.

  • The read/write-head may be moved depending on the move of the rule.
    move 'L indicates a move to the left.
    move 'R indicates a move to the right.
    move 'N indicates that that the read-write-head is not moved.

  • When no applying rule can be found, the procedure emulating the Turing-machine halts by raising an exception.

parameter

(Turing-report) → boolean?

(Turing-report on/off) → void?
  on/off : any/c
If on/off is not #f, the new value is #t. The initial value is #f. When the value of the parameter is true, a Turing-machine reports each move on the current output-port. Each line of the report shows:

  • The move counter, starting from 1.

  • The old-state of the control unit.

  • The new-state of the control unit, possibly the same as the old-state.

  • The tape-symbol (or empty-cell) encountered before replacing it.

  • The tape-symbol that is written, possibly the same one as just read.

  • The move of the read/write-head ('R, 'L or 'N).

  • The new position of the read/write-head and the new content of the tape shown as the result of (list head tail), the first element of the tail marking the new position of the read/write-head.

The report is best readable when the printed forms of internal-states do not vary much in their lengths. The same holds for tape-symbols.

parameter

(Turing-limit) → (or/c #f exact-positive-integer?)

(Turing-limit n) → void?
  n : any/c
When the parameter holds an exact-positive-integer?, say n, a Turing-machine halts with an exception when it does not reach a final-state within n or less moves. The initial value is #f. When n is not an exact-positive-integer?, the parameter is set to hold #f.

parameter

(Turing-move-count-width) → exact-nonnegative-integer?

(Turing-move-count-width n) → void?
  n : exact-nonnegative-integer?
In a report the move counter is padded with spaces such as to take n characters. Move counts requiring more characters are not truncated. The initial value is 0. For example, when expecting more than 9 but less than 100 moves, use (Turing-move-count-width 2) in order to produce a well alligned report with all move-counts having the same number of characters.

3 Running a Turing-machine

A procedure returned by make-Turing-machine, say Turing-machine, is called as follows:

procedure

(Turing-machine input) → 
nr-of-moves final-state output
  input : (listof tape-symbol)
Turing-machine : produced by procedure make-Turing-machine
nr-of-moves : exact-positive-integer?
output : (listof tape-symbol)

The output shows the content of the tape after the machine has reached a final-state, but without heading or trailing blanks or empty-cell.

If during the execution of the Turing-machine no rule can be found matching its current internal-state and the current element, the Turing-machine halts by raising an exception. For example:

> (require racket "make-Turing-machine.rkt")
> (define TM (make-Turing-machine
              'A '(T) 'E 'B '_
              '(((A x) (T _ N)))))
> (Turing-report #t)
> (TM '(y)) ; Error

......................................... initial tape: (() (y))

Turing-machine: no rule for state: A, with symbol: y

current content: (() (y))

Many Turing-machines never halt, sometimes depending on the rules only, sometimes depending on both the rules and the input. In some cases the problem can be predicted by looking at the rules or can be detected while a Turing-machine is running. However, because of the Halting Problem it is not possible to avoid the problem in all cases without putting a limit on the allowed number of moves. Procedure make-Turing-machine and the Turing-machines it produces do no checks at all against non-halting. For example, the following trivial Turing-machine clearly will loop forever with arbitrary input:

> (require racket "make-Turing-machine.rkt")
> (define TM (make-Turing-machine
              'A '(T) 'E 'B '_
              '(((A _) (A _ N)))))
> (Turing-report #t)
> (Turing-limit 9)
> (TM '(x))

......................................... initial tape: (() (x))

move 1, state A -> A, symbol x -> x, move: N, new tape: (() (x))

move 2, state A -> A, symbol x -> x, move: N, new tape: (() (x))

move 3, state A -> A, symbol x -> x, move: N, new tape: (() (x))

move 4, state A -> A, symbol x -> x, move: N, new tape: (() (x))

move 5, state A -> A, symbol x -> x, move: N, new tape: (() (x))

move 6, state A -> A, symbol x -> x, move: N, new tape: (() (x))

move 7, state A -> A, symbol x -> x, move: N, new tape: (() (x))

move 8, state A -> A, symbol x -> x, move: N, new tape: (() (x))

move 9, state A -> A, symbol x -> x, move: N, new tape: (() (x))

Turing-machine: no final state within max nr of moves

move-counter: 9

current state: A

current content: (() (x))

In this example rule '((A _) (A _ N)) alone already is easily understood to give rise to an infinite loop. While the TM is running, its state (content of the tape and the position of the read/write-head included) remains the same. Procedure make-Turing-machine could be adapted such as to predict the infinite loop just by checking the rules. It also could be adapted such as to produce a Turing-machine that can detect a repeated state. These adaptations have not been made, for the Halting Problem implies that there always remain cases in which a non-halting case cannot be predicted by procedure make-Turing-machine and cannot be detected while a Turing-machine is running.

4.1 Erase elements

The following Turing-machine always halts. A correct input is (x ... [+ x ...] ...), where the square brackets indicate an optional part of the input. The result of a correct input is the input without +. This is like addition of zero, one or more natural numbers, where natural number n is written as ‘x ...’ with n xs. With a correct input the machine halts in internal-state T. With incorrect input the machine halts in internal-state F. The machine never moves left of the start of the input. It moves exactly once to the right of the content of the tape. The resulting empty-cell is immediately replaced by a blank. Letter E is the empty-cell, B the blank and _ the dummy.

> (require racket "make-Turing-machine.rkt")
; 
> (define rules
  '(;  State 0 : Inspect the very first element.
    ;  Mark start x with y or start + with p.
    ;  This way we can detect the start of the input
    ;  when moving left and avoid moving left
    ;  of the start of the input.
    ((0 x) (1 y R))  ; Ok, go check the remainder of the input.
    ((0 +) (1 p R))  ; Ok, go check the remainder of the input.
    ((0 E) (T B N))  ; Empty input accepted.
    ((0 _) (F _ N))  ; Reject incorrect input.
    ;  State 1 : Check the remainder of the input.
    ((1 x) (1 x R))  ; Ok, continue the check.
    ((1 +) (1 + R))  ; Ok, continue the check.
    ((1 E) (2 B L))  ; Input is ok. Start the addition.
    ((1 _) (F _ N))  ; Reject incorrect input.
    ;  State 2 : Do the addition from right to left.
    ;  The content starts with y or p and ends with B.
    ((2 x) (2 x L))  ; Leave x as it is and continue addition.
    ((2 y) (T x N))  ; Start of input reached. Done.
    ((2 +) (3 x R))  ; Replace + by x and go replacing the last x by a blank.
    ((2 p) (3 y R))  ; Replace p by y and go replacing the last x by a blank.
    ;  State 3 : Go to end of tape.
    ((3 x) (3 x R))  ; Keep looking for end of input.
    ((3 B) (4 B L))  ; End of input reached. Replace one x by a blank.
    ;  State 4 : Replace the last x (or the y if there is no x) by a blank.
    ((4 x) (2 B L))  ; Replace x by a blank and continue addition.
    ((4 y) (T B N))  ; Replace y by a blank and accept.))
; 
> (define Turing-machine (make-Turing-machine '0 '(T F) 'E 'B '_ rules))

4.2 Binary addition

The following Turing-machine adds two natural numbers written in binary notation. The machine halts with every arbitrary input. A correct input is defined as follows:

input   = (operand + operand)

operand = bit bit ...

bit     = 0 | 1

An incorrect input yields final state F. A correct input yields final state T and output (bit bit ...) showing the sum of the two operands. More precisely the output is (1 bit ...) or (0), id est, without leading zeros. Letter E is the empty-cell, B the blank and _ the dummy. The initial content of the tape is modified such as to result in the sum. In the sum a 0 bit is written as x and a 1 bit as y. During the addition the content of the tape is (bit-0-or-1 ... x-or-y ... + bit-0-or-1 ...). Bits of the second operand are processed starting from the least significant one. Every such bit is replaced by a blank before it is processed. The significance of the blanked bit is the same a that of the right-most bit-0-or-1 of the first operand. After all bits of the second operand have been processed, the + is removed, x and y are reverted to 0 and 1 and leading zeros are removed.

4.3 Decimal addition

The following machine expects ((n m) ...) as input, where each n and each m is an exact integer between 0 (inclusive) and 10 (exclusive).
